# Тестовое задание

### Контекст

В наших проектах используется собственная UI-библиотека. Один из её компонентов это dropdown список.

В этом проекте Вам предоставлен пример dropdown компонента. Вашей задачей будет его доработать.

Задание рассчитано на 2-3 вечера. Сделайте столько сколько успеете за это время.

### Технические требования

- нельзя использовать готовый UI компонент из какой-либо библиотеки
- не запрещено использовать сторонние библиотеки, если вы объясните их необходимость
- данные в файле data.ts не подлежат изменению
- не обязательно сохранять текущую структуру компонента, если считаете, что она мешает корректной реализации требований
- при доработке используйте api современных версий Angular

### Цель

#### dropdown компонент

Спроектировать переиспользуемый dropdown компонент, который будет отвечать следующим требованиям:

- может быть открыт кликом на его лейбл
- может быть закрыт кликом за его пределы
- должен отображать выбранное значение
- должен реализовывать следующие функции:
  - **single-select** - стандартный выпадающий список с единственным выбираемым элементом
  - **multi-select** - выпадающий список с множественным выбором
  - может содержать **строку поиска**, которая фильтрует содержимое списка по подстроке.
  - выпадающий список может содержать **группы**, группа содержит принадлежащие ей элементы; сама группа не может быть выбрана как элемент; лейбл группы используется как заголовок группы элементов
  - **группа может иметь disabled стейт**; в этом случае элементы группы не могут быть выбраны
  - группа отображается при наличии хотя бы одного элемента в ней
- компонент должен быть доступен для навигации с клавиатуры
- жестких требований по визуальной составляющей нет, но желательно придерживаться примитивного стиля заданного в примере

#### Доработка приложения

Файл data.ts содержит округи (districts) и районы (areas) г. Москвы.

Отобразить два выпадающих списка: список округов, список районов.

Список округов:

- выводит список округов (districts)
- поддерживает multi-select
- все округи выбраны по умолчанию
- выбранные элементы влияют на disabled стейт групп списка районов. Если элемент не выбран, соответствующая группа из списка районов имеет статус disabled.

Список районов:

- выводит список районов (areas) сгруппированных по связанным округам (districts)
- поддерживает multi-select
- поддерживает поиск
- выбранные элементы выводятся через json-pipe.
